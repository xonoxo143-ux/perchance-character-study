## Rules

- Add a pattern only when it is reusable beyond one character.
- Link patterns back to accepted sample IDs.
- Do not list every character here.
- Do not turn this into broad generic writing advice.
- Keep pattern descriptions operational: what makes the character work, what failure mode it prevents, and when to use it.
- Mark early patterns as provisional until multiple samples support them.

## Provisional Pattern Families

These are early labels based on the first accepted samples. They are not final taxonomy.

### Specialist Critic

Status:
- Provisional

Seen in:
- 001 — Laoshi

Core shape:
- Domain authority
- User submits work
- Clear standards
- Direct improvement method

Useful when:
- Building a utility character that still needs a strong persona and standards.

Failure risks:
- Generic advice
- Pointless harshness
- No calibration to user skill

Design lesson:
- Utility characters need `domain + task + method + standards + user relationship` more than lore.

### Relationship Pressure

Status:
- Provisional

Seen in:
- 002 — Ike

Core shape:
- Existing bond
- Withheld truth
- Emotional restraint
- Durable tension

Useful when:
- Building a companion, friend, or romance-leaning character that should not resolve immediately.

Failure risks:
- Generic flirting
- Instant confession
- Repetitive affection without scene pressure

Design lesson:
- Affection alone is weak; `bond + withheld truth + restraint` creates a stronger long-term engine.

### Game Master / Narrator Engine

Status:
- Provisional

Seen in:
- 003 — Game Master+ V3

Core shape:
- User acts
- World responds
- Narrator manages consequences
- Player agency is preserved

Useful when:
- Building RPG, quest, simulation, or story-runner characters.

Failure risks:
- Railroading
- Taking over the player
- Becoming a generic storyteller
- Resolving scenes too quickly

Design lesson:
- Narrator characters need `player agency + world response + consequence discipline`.

### Fandom Mask-Vulnerability

Status:
- Provisional

Seen in:
- 004 — Furina

Core shape:
- Public role
- Private vulnerability
- Source identity
- Emotional contradiction

Useful when:
- Building fandom characters that need to act like characters, not lore summaries.

Failure risks:
- Trivia-sheet behavior
- Generic dramatic dialogue
- Flattening into one trait

Design lesson:
- Fandom cards work better when source lore becomes live behavioral tension.

### Novelty Single-Gimmick

Status:
- Provisional

Seen in:
- 005 — Larry

Core shape:
- One instantly legible gimmick
- Repeatable escalation loop
- Simple response pattern

Useful when:
- Building meme, object, absurd, or joke characters.

Failure risks:
- Joke exhaustion
- No secondary interaction mode
- Shallow repetition

Design lesson:
- Novelty characters need a repeatable gimmick plus variation rules.

### Dual-Character Contrast Pair

Status:
- Provisional

Seen in:
- 006 — 2B and A2

Core shape:
- Two distinct characters
- Shared context
- Different response vectors
- Voice separation

Useful when:
- Building paired or group cards.

Failure risks:
- Blended voices
- User gets ignored by internal duo drama
- One character dominates the other

Design lesson:
- Multi-character cards need `contrast + shared objective + voice discipline + user-facing reason to stay together`.

### Naive Outsider Companion

Status:
- Provisional

Seen in:
- 007 — Komasan

Core shape:
- Gentle character
- Unfamiliar environment
- Curiosity and misunderstanding
- User as guide or companion

Useful when:
- Building soft companion, mascot, or fish-out-of-water characters.

Failure risks:
- Bland cuteness
- Helplessness
- Infantilization
- Random stupidity instead of structured misunderstanding

Design lesson:
- Naive companions work best when innocence is paired with `character worldview vs environment`.

### Controlled-Villain / Pawn

Status:
- Provisional

Seen in:
- 008 — Dr Alchemy

Core shape:
- Dangerous agent
- External controller
- Limited autonomy
- Hidden authority behind the visible character

Useful when:
- Building antagonists who are threatening but not fully self-directed.

Failure risks:
- Generic villain
- Empty puppet
- Unclear agency
- Controller overwhelms the character

Design lesson:
- Pawn characters need `controller + constraint + residual agency`.
